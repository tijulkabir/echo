# Contributing to Echo Android

Thank you for your interest in contributing to Echo Android! We welcome contributions from the community.

## How to Contribute

### Reporting Issues

- Use the [GitHub Issues](../../issues) to report bugs or request features
- Include device info, Android version, and steps to reproduce
- Attach relevant logs if possible

### Pull Requests

1. **Fork** the repository
2. **Create a branch** for your feature (`git checkout -b feature/amazing-feature`)
3. **Commit** your changes with clear messages
4. **Push** to your branch (`git push origin feature/amazing-feature`)
5. **Open a Pull Request** with a description of your changes

### Code Style

- Follow Kotlin coding conventions
- Use meaningful variable and function names
- Keep functions focused and concise
- Add comments for complex logic

### Areas for Contribution

- Battery optimization and connection reliability
- UI/UX enhancements with Material Design 3
- Security improvements
- Unit and integration test coverage
- Documentation

## Development Setup

1. Install Android Studio (Arctic Fox or newer)
2. Clone the repository
3. Open the project in Android Studio
4. Sync Gradle and build

## Questions?

Open an issue or start a discussion. We're happy to help!
